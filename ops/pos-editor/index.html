📖 POS Product Code Legend

Decode any POS code on this page without leaving.

Code Anatomy — 3 segments

{BRAND}-{SHAPE}-{ITEM}
BRANDNCH / HE / HN — outlet that sells this product
SHAPES1...S10 — recipe pattern
ITEM3-char abbreviation, unique within (BRAND,SHAPE)

The POS code IS the operation. It encodes which outlet sells the item, which recipe shape governs RM consumption, and the unique short identifier — three coordinates, one canonical row, no free-text interpretation step between data and action.

BRAND values · selling outlet

CodeMeaningExamples
NCH Nawabi Chai House — Irani chai cafe. Concentrated menu (chai + haleem + snacks). NCH-S5-IRC Irani Chai · NCH-S5-BMK Bun Maska
HE Hamza Express — QSR biryani. ~176 SKU menu, Dakhni cuisine focus. HE-S3-CKB Chicken Biryani · HE-S6-TCK Tandoori Chicken
HN HN Hotels — cross-brand items shared across both outlets (drinks, packaging, etc.). HN-S10-PKG Packaging fee · HN-S1-WTR Bottled water

SHAPE values · recipe pattern

ShapePatternReal example
S1 RESALE. Bought-and-sold-as-is. No recipe transformation. Direct pass-through from RM to POS. NCH-S1-NLF (Niloufer 100g pack)
S2 MULTI_UNIT. N units of the same RM per sale. Counted, not measured. NCH-S2-OB3 (Osmania Biscuit · Pack of 3)
S3 SINGLE_PORTION. Exact recipe with multiple RMs in fixed proportions. One sale = one portion of recipe. NCH-S3-HLM (Haleem · 1 plate)
S4 MULTI_PORTION. Recipe scaled to multiple portions sold as a unit (e.g. half-kg / full-kg). NCH-S4-HLF (Haleem · half-kg / family pack)
S5 BATCH+APX. Drawn from a prepared batch with approximate per-cup ratios. Per-sale RM consumption is a ratio of batch size. NCH-S5-IRC (Irani Chai — drawn from chai vat)
S6 MARINATED_PROTEIN. Protein with marination prep. HE-side. Sale unit = marinated portion. HE-S6-TCK (Tandoori Chicken)
S7 WRAP. Wrap / roll item. HE-side. Sale unit = one wrap with bread + filling + sauces. HE-S7-CKR (Chicken Roll)
S8 UNIT_PROTEIN. Unit-priced protein item. HE-side. Sale unit = one piece (kabab, leg piece). HE-S8-CKK (Chicken Kabab)
S9 COMPOSITE. Composite of multiple sub-items sold together (thali plate, combo). HE-S9-THL (Thali plate)
S10 NO_RM. No RM consumption — service charge, packaging fee, delivery fee. Pure revenue line. HN-S10-PKG (Packaging fee)

Each SHAPE governs how RMs are consumed when the POS row sells. The future Action ring (rm_recipes) will reference pos_code by FK, with the SHAPE constraining the recipe structure: S1 needs a single RM ref, S3 needs a recipe with proportional RMs, S5 needs a batch-ratio table, etc.

ITEM rules

RuleDescription
LengthExactly 3 characters, uppercase A-Z and 0-9 only
SourceAuto-derived from pos_name (consonants stripped of vowels), owner-overridable
UniquenessUnique within (BRAND, SHAPE) — NCH-S5-IRC blocks any other NCH-S5-IRC, but HE-S5-IRC would still be allowed.
CaseAlways uppercase. Editor normalises lowercase on save.

Forbidden combinations

ForbiddenWhy caught
BRAND outside {NCH, HE, HN}Enumerated set. Anything else rejected.
SHAPE outside {S1...S10}Enumerated set. S99, SX etc. rejected.
ITEM not exactly 3 charsLength is fixed; IRA ok, IRANI rejected (too long).
ITEM with lowercase or symbolsUppercase A-Z / 0-9 only; irc normalised to IRC; IR- rejected.
Duplicate (BRAND, SHAPE, ITEM)Returns 409 with the existing pos_code.

Why this entity ring closes Framework Gap 2

Before this editor, the 33 NCH service POS items + 21 retail Niloufer SKUs lived only in the Foundation Sheet's "POS Products" tab. Recipes referenced POS codes by string with no FK enforcement — when a POS code mutated (size variant added → renamed), recipes silently went stale.

Now the entity ring is closed: pos_code is the canonical key, the constraint set excludes impossible (BRAND, SHAPE, ITEM) combinations at write time, and the future rm_recipes table can FK against pos_code — so renaming a POS item raises a referential error instead of silently breaking recipes.

Per COA Doctrine: Entity → Action → Event. POS Products = entity. Recipes (S3 ratios, S5 batch ratios, etc.) = action. Sales events = composition. This ring closes before the next opens.
