Equations · The Mathematical Spine

EQ_PROMISE_PILE(RNR_X) =
  Σ pos.payment WHERE pm_id ∈ {PM_48, PM_40} AND partner_id = RNR_X
  − Σ razorpay.payment WHERE qr_id = qr_for(RNR_X)
  − CCR_X − cash_settled(X)
  ⇒ MUST = 0 at shift close
EQ_AGG_NET_EXPECTED =
  Σ pos.order WHERE config = POS_29 ∧ pm_id = PM_46 (Swiggy)
  × (1 − 0.246 ± 0.005)
  ⇒ match BNK_HDFC_OPS credit T+contract days
EQ_RZP_PAYOUT_EXPECTED =
  Σ razorpay.payment(d) − Razorpay fees
  ⇒ match BNK_HDFC_MAIN credit on (d+1)
EQ_CASH_PILE_NCH =
  opening_float
  + Σ cash_in (PM_37 sales + runner cash_settled + handovers_in)
  − Σ cash_out (counter expenses + handovers_out)
  ⇒ MUST = physical drawer count at shift close
EQ_LIVE_INVENTORY(item) =
  closing_count(yesterday)
  + Σ purchase.receive(item, today)
  − Σ pos.sale × BoM(item, product) (live)
  − Σ wastage(item, today)

Claude Agent Execution Layer FUTURE · cron-driven

AG_INVENTORY_WATCHDOG
Cron: every 5 min · Reads live POS sales (Odoo JSON-RPC) · BoM-decomposes into items · Computes EQ_LIVE_INVENTORY(item) for every definitive item · Fires WABA template to Basheer when item drops below threshold.
cron: */5 * * * * · template: ops_alert_v1 · scope: NCH definitive items
AG_RZP_POS_RECON
Cron: every 5 min · Pulls Razorpay payments since last cursor · For each, checks pos.payment exists within ±10 min · If unmatched, raises RULE_G5 alert.
cron: */5 * * * * · template: rp_no_pos_v1 · scope: 7 RPQR codes
AG_PROMISE_CLOSURE
Cron: every 30 min · Per RNR, computes EQ_PROMISE_PILE · If any has been > 6h non-zero, fires WARN; > 24h fires ERROR.
cron: */30 * * * * · template: ops_followup_v1 · scope: RNR_64–RNR_68
AG_BANK_RECON
Cron: 00:30 IST daily · For each bank instrument, computes EQ_RZP_PAYOUT_EXPECTED + aggregator nets · Compares to actual HDFC/Federal credits · Drift > ₹100 = ERROR.
cron: 30 0 * * * · template: hdfc_eod_drift_v1 · scope: BNK_HDFC_MAIN, BNK_HDFC_OPS
AG_CASH_RECONCILE
Triggered on shift close in /ops/v2 · Compares physical drawer count to EQ_CASH_PILE_NCH · Drift > ₹500 attributes to closing PIN as discrepancy.
trigger: shift_close · template: cash_drift_v1 · scope: NCH counter pile
AG_DIGEST_MORNING
Cron: 09:00 IST daily · Summarises yesterday's resolved + pending alerts + manager ack times · Fires ops_digest_v1 to Nihaf.
cron: 0 9 * * * · template: ops_digest_v1 · scope: events_bus 24h
Architecture:
Every agent is a Cloudflare Worker triggered by cron. They use the Anthropic Claude API (key on CF secrets) to evaluate complex variance situations, call /api/comms?action=send with the right template + tier, and write results to events_bus. The owner only sees breaches Basheer didn't close in time. Manager efficiency is tracked in comms_outbox.acked_at − sent_at per ack_action.
