Canonical Code Recipe

{PAY_SEQ}-{SELLS}-{OPM_LETTER}-{PMS}-{IDENTITY}
  • PAY_SEQ: Rf (recurring) or Pf (prepaid)
  • SELLS: L (loose) · B (branded) · LB (both)
  • OPM_LETTER: M if any M-* sub-channel · A if any A-* sub-channel. Sub-channel (PHONE/WA/WI/RT/AP/WB/API) becomes a property in the vendor record, not in the code.
  • PMS: C (cash) · D (digital) · H (hybrid)
  • IDENTITY: 3-10 char vendor abbreviation (uppercase)
Worked example: Prabhu Buffalo Milk Vendor with hand-filled OPM=M-WA, PMS=C → canonical code: Rf-L-M-C-PRABHU
