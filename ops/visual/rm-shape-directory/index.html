SECTION 2

Code Anatomy — 5 segments, left to right

{BRAND} - {TYPE} - {USAGE} - {SOURCING} - {ITEM}
BRAND
Position 1 · 2-3 chars
Which outlet this RM belongs to, or whether it crosses brands under the HN parent.
3 values · NCH · HE · HN
TYPE
Position 2 · 2 chars
Approximation × Use-scope. How the item is measured, and how many products consume it.
4 values · AM · AS · DM · DS
USAGE
Position 3 · 1-3 chars
Where this RM gets consumed — production recipes, retail resale, or operational fuel.
15 valid profiles · primary uppercase
SOURCING
Position 4 · 1-3 chars
How the RM enters the outlet — loose vendor, branded SKU, or in-house production.
15 valid profiles · primary uppercase
ITEM
Position 5 · exactly 3 chars
3-letter abbreviation derived from the RM name. Owner-overridable. Unique within (BRAND, TYPE).
e.g. SUG · MLK · LPG · OSB
Reading order: Brand owns the outlet. Type owns the math. Usage owns the consumption flow. Sourcing owns the supply flow. Item owns identity. Every code is a coordinate, never a description.
SECTION 3

BRAND prefix directory Position 1

Which entity owns this raw material — an operating outlet, or the HN parent declaring cross-brand ownership.

CodeMeaningReal example
NCH Nawabi Chai House — outlet-specific RM, only consumed at NCH. NCH-AM-P-Lb-MLK Buffalo Milk
HE Hamza Express — outlet-specific RM, only consumed at HE. (none in editor yet — reserved for HE rollout)
HN Cross-brand — used at both NCH AND HE under the HN parent umbrella. HN-AM-P-Lb-SUG Sugar
Important: HN itself has no consumption — it's the parent declaring "this RM is shared across the operating brands beneath it." A purchase booked under HN-… is allocated to NCH and HE at the action layer.
SECTION 4

TYPE directory Position 2 · 2 chars

A 2-character code: {Approximation}{Use-scope}. Tells the system how the item is measured AND how many products consume it.

1st char — Approximation

LetterMeaningUse when
AApproximate — portioned by weight/volume, not 1=1.Items measured in g, kg, ml, L (sugar, milk, oil, tea powder).
DDefinitive — counted exactly, 1 unit = 1 unit.Items in discrete pieces, packets, cylinders (biscuit pack, LPG cylinder, coffee pouch).

2nd char — Use scope

LetterMeaningUse when
MMulti-use — consumed by 2+ POS products.Most ingredients (sugar, milk, butter, biscuit consumed across many recipes).
SSingle-use — consumed by exactly 1 POS product.Specific patties, single-recipe items (chicken cutlet raw → chicken cutlet only).

Where this RM gets consumed. Letters: P Production · R Retail · O Operational.

Letters

Letters

SECTION 7

ITEM abbreviation directory Position 5 · exactly 3 chars

A 3-letter, all-uppercase abbreviation that gives the RM its identity within the (BRAND, TYPE) tuple.

Length

Exactly 3 characters, uppercase A-Z. Always 3 — never 2, never 4.

Source

Auto-derived from RM name (first 3 unique consonants, vowels as fallback). Owner override allowed in editor.

Uniqueness

Must be unique within (BRAND, TYPE) tuple. Two NCH-AM RMs cannot both be MLK; one across NCH-AM and HN-AM is allowed.

Case

Always uppercase. The editor normalises lowercase to uppercase on save.

SECTION 10

Worked Examples — Reading codes left to right

Six real codes, walked through one segment at a time.

1 · HN-AM-P-Lb-SUG Sugar

HN-AM-P-Lb-SUG │ │ │ │ └─ Sugar (item identity) │ │ │ └────── Loose primary, branded alternate (loose sugar dominant; Madhur 1kg also stocked) │ │ └────────── Production only (consumed by recipes; not sold as retail) │ └────────────── Approximate Multi-use (weighed in kg; used by tea, snacks, biscuits) └────────────────── HN — cross-brand, used at both NCH AND HE
Reads as: "An HN-shared Sugar RM, approximate-multi-use, production-only, primarily sourced loose with a branded alternate available."

2 · NCH-AM-Pr-Bl-TEA Tea Powder

NCH-AM-Pr-Bl-TEA │ │ │ │ └─ Tea Powder (item identity) │ │ │ └────── Branded primary, loose alternate (Wagh Bakri / Society as primary; loose tea fallback) │ │ └────────── Production primary, retail alternate (mostly brewed; occasionally sold as packs) │ └────────────── Approximate Multi-use (weighed; used across all chai variants) └────────────────── NCH — Nawabi Chai House outlet only
Reads as: "An NCH Tea Powder RM, approximate-multi-use, primarily a production input but with a retail-resale path, primarily branded with loose as fallback. The single most consumed AM-RM at NCH."

3 · HN-AM-O-B-LPG LPG

HN-AM-O-B-LPG │ │ │ │ └─ LPG (item identity — cooking gas) │ │ │ └────── Branded only (Indane / HP cylinder, no loose sourcing path) │ │ └────────── Operational only (fuels the kitchen — not in any recipe, not sold) │ └────────────── Approximate Multi-use (consumed by every burner, measured by cylinder weight) └────────────────── HN — cross-brand, both outlets cook on LPG
Reads as: "A cross-brand LPG RM, approximate-multi-use, operational consumable (no recipe, no resale), branded-only sourcing — the canonical example of the USAGE=O profile."

4 · NCH-DM-R-B-FRB Fruit Biscuit (Niloufer)

NCH-DM-R-B-FRB │ │ │ │ └─ Fruit Biscuit (item identity — Niloufer brand) │ │ │ └────── Branded only (Niloufer SKU pack — no loose path, no in-house production) │ │ └────────── Retail only (sold as-is to customers; not used in any NCH recipe) │ └────────────── Definitive Multi-use (counted in packs; multiple SKU sizes 100g/250g/500g/1kg) └────────────────── NCH — sold only at NCH counter
Reads as: "An NCH Fruit Biscuit RM, definitive-multi-use, retail only, branded only — a pure resale SKU. Multiple pack-sizes give it the M (multi-use) attribute via SKU multiplicity, not recipe multiplicity."

5 · NCH-AS-P-Li-CCT Chicken Cutlet Raw

NCH-AS-P-Li-CCT │ │ │ │ └─ Chicken Cutlet Raw (item identity) │ │ │ └────── Loose primary, in-house alternate (vendor-supplied raw cutlet primary; in-house batch fallback) │ │ └────────── Production only (only goes into Chicken Cutlet POS item) │ └────────────── Approximate Single-use (weighed; consumed by exactly 1 POS product) └────────────────── NCH — NCH outlet only
Reads as: "An NCH Chicken Cutlet Raw RM, approximate-single-use, production-only, primarily loose-sourced with an in-house production fallback. The textbook example of an Li sourcing profile."

6 · NCH-DS-P-B-CPP Coffee Pouch

NCH-DS-P-B-CPP │ │ │ │ └─ Coffee Pouch (item identity — single-serve sachet) │ │ │ └────── Branded only (Bru / Nescafe pouch SKU) │ │ └────────── Production only (goes into Nawabi Coffee preparation) │ └────────────── Definitive Single-use (counted in pouches; consumed by exactly 1 POS product) └────────────────── NCH — NCH outlet only
Reads as: "An NCH Coffee Pouch RM, definitive-single-use, production only, branded only — 1 pouch = 1 cup of Nawabi Coffee, no other use."
SECTION 11

Connection to other layers

Action layer

Purchases, productions, transfers — built on top of these RM codes, adding V-shape (payment behaviour) and supplier dimensions.

Event layer

Reads RM codes + Actions to compose business events (sales, COGS, expenses) deterministically.

Doctrine

All three rings — Entity (this page), Action, Event — are formalised at /ops/visual/coa-doctrine/.

The RM canonical code is the foundation. Every Action references one RM. Every Event aggregates Actions. Get the shape right here, and the rest of the system is dumb-proof.
