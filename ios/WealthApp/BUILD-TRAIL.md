## Architecture decisions (do NOT relitigate)
1. **Standalone native app**, NOT a Hukum chamber (owner boundary: keep separate from Chat 33 Voice / Chat 34 App).
2. **Logic stays server-side.** The app renders state + posts intent; it decides nothing.
3. **Auth = local PIN unlock + embedded DASHBOARD_KEY.** The PIN guards the surface on-device; the key authorizes the API. The key gates BOTH reads and order execution â€” treat it as order-capable.
4. **Mode banner fails UNSAFE.** If config can't be read â†’ "MODE UNKNOWN", never "SAFE". Shows the live `block_real_orders` + `engine_mode` values. (Caught a false-"SAFE" bug by verifying against live config â€” config was the slowest of 6 reads; fix = load config first.)
5. **Orders forced to MIS** (intraday auto-square), never CNC. Idempotency tag `HN_WE_IOS_<ts>` + button disabled while placing.
6. **Practice mode default-on** so the flow can be rehearsed with zero risk / no Kite.
